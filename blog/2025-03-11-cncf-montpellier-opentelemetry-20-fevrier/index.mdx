## Une immersion dans l'écosystème Cloud Native 🌍

Le 20 février, j’ai eu l’occasion de participer à la Cloud Native CNCF de Montpellier, un événement dédié aux technologies cloud-native et à l'écosystème CNCF (Cloud Native Computing Foundation). Entre discussions passionnantes et démonstrations techniques, une découverte m’a particulièrement marqué : **OpenTelemetry**.

## OpenTelemetry, c'est quoi ? 🤔

OpenTelemetry est un projet open source sous l’égide de la CNCF qui fournit un ensemble d’outils, d’API et de SDK pour instrumenter, collecter et exporter des données de télémétrie (logs, metrics, traces) afin de mieux observer et diagnostiquer les applications distribuées.

En d’autres termes, c’est une boîte à outils permettant d’avoir une vision claire et détaillée de ce qui se passe dans un système complexe, sans être enfermé dans un outil propriétaire.

<ArialInArticle />

## Pourquoi s'y intéresser ? 🚀

Lors des différentes conférences et échanges, plusieurs points ont retenu mon attention :

-   **Standardisation** : OpenTelemetry propose un standard unifié pour la collecte de données d’observabilité, réduisant ainsi la dépendance aux solutions propriétaires comme Datadog, New Relic ou AppDynamics.
-   **Interopérabilité** : Compatible avec des langages variés (TypeScript, Go, Python, Java...) et des outils comme Prometheus, Jaeger et Grafana.
-   **Flexibilité** : Il s’intègre aussi bien avec des architectures microservices qu’avec des monolithes.

## Une démonstration convaincante 🎯

Pendant la conférence, une démo en live a illustré comment instrumenter une application Node.js avec OpenTelemetry. Voici les grandes étapes :

1. **Installation du SDK** :
    ```sh
    npm install @opentelemetry/sdk-node @opentelemetry/api
    ```
2. **Ajout de la télémétrie aux requêtes HTTP** :
    ```javascript
    const {NodeTracerProvider} = require('@opentelemetry/sdk-trace-node');
    const provider = new NodeTracerProvider();
    provider.register();
    ```
3. **Exportation vers Jaeger pour visualisation** :
    ```sh
    docker run -d --name jaeger -p 16686:16686 -p 14268:14268 jaegertracing/all-in-one:latest
    ```
